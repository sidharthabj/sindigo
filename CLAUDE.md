# Sindigo

A social reading tracker. Users maintain three shelves (Wishlist, Currently Reading, Read), write notes/reviews, rate books out of 5 (displayed as book icons), follow friends, and see an activity feed.

## Spec & Plan

- Full design spec: `docs/superpowers/specs/2026-06-06-sindigo-design.md`
- Implementation plan (phases + tasks): `PLAN.md` — check the Project Status table at the top to see which phase is current

## Tech Stack

- **Framework:** Next.js 15 App Router
- **Database + Auth:** Supabase (PostgreSQL + OAuth)
- **DB client:** Supabase TypeScript client with generated types (`lib/types/database.ts`)
- **UI components:** shadcn/ui
- **Animation:** Framer Motion
- **Styling:** Tailwind CSS
- **Book data:** Google Books API
- **Hosting:** Vercel

## Key Conventions

- **Supabase clients:** Use `lib/supabase/server.ts` in Server Components and Server Actions; `lib/supabase/client.ts` in Client Components only.
- **Server Actions** live in `lib/actions/` — one file per domain (books, follows, likes, comments). Always call `revalidatePath` after mutations.
- **Types:** Never edit `lib/types/database.ts` — it is generated by Supabase CLI. App-level types go in `lib/types/index.ts`.
- **Book rating:** 1–5 integer stored in DB, rendered as book icons (not stars) via the `Rating` component.
- **Note field:** The `shelf_entries.note` column is the single text field for all statuses — labeled "Why I want to read this" (wishlist), "My thoughts so far" (reading), or "My review" (read).
- **Routes:** Profile at `/[username]` (not `/profile/[username]`). Reserved slugs: `feed`, `search`, `settings`, `login`, `signup`.
- **Book detail:** `/[username]/books/[bookId]` where `bookId` is the canonical `books.id` UUID.
- **Activity generation:** Emitted in Server Actions, not DB triggers. Rule: if user writes a note at the same time as marking read, only emit `finished_book` (not `wrote_review`).
